# tasks/lessons.md

Gotchas from prior sessions. Grep before fighting a regression.

## Stack / scaffold
- Next **16.2.9**, React **19.2.4**, **Tailwind v4** (CSS-first `@theme` in `globals.css` — no `tailwind.config.js`). Design tokens live in `@theme`, mirroring `DESIGN.md`.
- `create-next-app` refuses to scaffold into a non-empty dir; we scaffolded in a temp dir and rsync'd in, excluding the generated `CLAUDE.md`/`AGENTS.md` so our docs survived.

## Conventions to remember
- **No web fonts.** `system-ui` only (low-bandwidth). Don't reintroduce `next/font`.
- **Color never alone** — every report type = color + icon + Spanish label.
- Use a type's `-text` token for text on canvas; `-fill` is for backgrounds only (amber/slate fills fail text contrast).
- `"use client"` only when a component owns state/refs/browser APIs. Skeletons, forms, map, outbox = client; feed/detail = RSC.

## Boneyard
- `BoneSkeleton` needs the SSR mount-guard (boneyard's client-only `data-boneyard-content` breaks hydration). Don't remove it.
- `src/bones/registry.ts` is generated by `pnpm bones:build` — never hand-edit. It needs a running dev server + headless Chrome (Playwright); not run yet in Phase 0.

## Serwist
- SW is **disabled in development** (`next.config.ts`) so stale caches never mask fresh code. It builds + registers for production only.
- **`@serwist/next` is webpack-only and breaks Next 16 Turbopack builds** ("webpack config with no turbopack config"). We build/dev with `next ... --webpack`. Don't switch back to Turbopack unless migrating to stable `@serwist/turbopack`.
- `src/app/sw.ts` needs `/// <reference lib="webworker" />` at the top or TS can't find `ServiceWorkerGlobalScope`.
- Generated `public/sw.js` must be in eslint `globalIgnores`, else ~100 lint warnings from the minified worker.

## Environment
- **`/home` runs near-full on this machine** (87G/92G). A `next build` regenerates ~130M of `.next` and will ENOSPC if free space is low. If a write fails with ENOSPC: `rm -rf .next` (regenerable) for headroom; prefer `tsc --noEmit` (writes nothing) to typecheck under disk pressure.

## Phase 1 patterns
- Offline-first = form writes to **IndexedDB (Dexie outbox)**, NOT a server action. Sync is a separate `flushOutbox()` triggered on mount/online/foreground.
- Supabase is **pluggable via env** — `getSupabase()` returns null when unset; app still queues reports. Never hard-require it.
- PostGIS insert from PostgREST: send location as EWKT text `SRID=4326;POINT(lng lat)`.
- Idempotent sync: `upsert(..., {onConflict:'client_uuid', ignoreDuplicates:true})` + Storage `upsert:false` tolerating "already exists" → RLS needs only INSERT policies.

## Phase 2 / maps
- **MapLibre must be dynamically imported** inside MapView's effect (`await import("maplibre-gl")`), and map-style.ts uses a **type-only** maplibre import — otherwise SSR evaluates `window`-touching code and crashes. No `next/dynamic ssr:false` needed this way.
- `@types/geojson` must be a direct devDependency — pnpm doesn't hoist maplibre's transitive copy for `import ... from "geojson"`.
- Register `pmtiles://` protocol once (`maplibregl.addProtocol`) before creating the map; gate cluster-count `symbol` layers on a real basemap (they need glyphs).
- Basemap is env-driven (`NEXT_PUBLIC_MAP_STYLE_URL`); blank fallback style still renders markers.

## React 19 / new react-hooks lint rules (Next 16)
- `react-hooks/refs`: **never assign `ref.current = x` during render.** Do it in an effect (`useEffect(() => { ref.current = x }, [x])`), or drop the ref and add the value to a `useCallback` dep.
- `react-hooks/set-state-in-effect`: flags any effect that calls a function containing `setState`, even behind an `await`. For genuine fetch-on-mount, restructure so setState runs post-await AND disable the rule on that one line with a justification comment.
- Stray `/home/zodx/package-lock.json` made Next infer the wrong workspace root → pin `outputFileTracingRoot: import.meta.dirname` in next.config.

## Dev server hygiene
- A previous `next dev` can keep port 3000 bound → next `pnpm dev` exits 1 ("Another next dev server is already running"). In smoke scripts: bound the readiness wait (don't `until curl` forever), check `kill -0 $PID`, and `pkill -f next-server` after.

## Phase 3 / Supabase moderation
- **Public-on-insert model:** status defaults `published`; insert RLS `with check (status='published')`. Reactive moderation, not a gate.
- **Postgres enum values can't be ADDed and USEd in the same transaction** → we set the final enum in `0001` rather than ALTERing it in `0002`.
- **Views that expose privileged columns must be `security_invoker = true`** so base-table RLS applies — otherwise a definer view leaks (e.g. contact_phone) to any authenticated user. `reports_public` (anon-safe columns) is a normal definer view; `reports_moderation` (has phone) is security_invoker so only moderators see rows.
- **Rate limiting via a BEFORE INSERT trigger**: derive IP from `current_setting('request.headers', true)::json->>'x-forwarded-for'`, store only a salted `digest()` hash (pgcrypto). Enforce windows by counting recent rows.
- **Moderator auth = Supabase Auth magic link + `moderators` allowlist + `is_moderator()`**; RLS enforces writes. The `/admin` UI gate is UX, never the security boundary. Set `getSupabase()` auth to `persistSession/detectSessionInUrl: true` or the magic-link redirect won't establish a session.
- Dedup: `pg_trgm similarity()` + PostGIS `st_dwithin` + time window in a `security definer` RPC, surfaced for manual merge (never auto-merge).

## Phase 4 / i18n + PWA
- **RSC-aware i18n without next-intl:** locale in a cookie; `getServerT()` (reads `cookies()`) for server components, `useT()` context for client. TypePin/StatusBadge are client islands so their labels are locale-correct even inside server pages — context flows to client components through an intervening server component.
- **Reading `cookies()` in the root layout forces dynamic rendering** of every page (○ static → ƒ dynamic). Fine here, but if static prerender matters, use `/[locale]` routing instead.
- Typed dictionary: `en: Record<keyof typeof es, string>` makes TS fail the build if a key is missing in either locale — cheap completeness guarantee.
- Localize `<title>` per page via `generateMetadata()` (async, uses `getServerT`), not a static `metadata` export.
- Serwist offline fallback: add `{ url: "/offline", revision: null }` to `precacheEntries` AND a `fallbacks.entries` matcher on `request.mode === "navigate"`.
- `@google/design.md lint DESIGN.md` is real and useful — 0 errors = contrast/refs OK; "defined but never referenced" warnings are just spec hygiene.

## Phase 4 / extraction + notes
- LLM pool ported from market-chat: round-robin (`pickTier1`) + in-memory circuit breaker. `server-only` imports break vitest — aliased to a stub in `vitest.config.ts`.
- Hybrid extraction: regex (cédula/phone/link, isomorphic, in `patterns.ts`) + LLM (names/addresses, `llm.ts`). Free models are sloppy → permissive zod schema, normalize after (`normalizeStrings`).
- Client can't reach the LLM (keys server-only). Offline-first means no server in the sync path, so extraction is a fire-and-forget `POST /api/extract { kind, clientUuid }` after sync / note insert. Idempotent on `extracted_at`; `?force=1` to redo.
- Notes are anonymous, keyed by client-generated `client_uuid`, rate-limited by the same ip_hash trigger pattern as reports (0002). Public reads go through `report_notes_public` to hide `ip_hash`.
- Cédula is full-public via `NEXT_PUBLIC_EXTRACT_CEDULA_FULL` (default true); `maskCedula` + `applyCedulaPolicy` in feed.ts flip it to masked in one env change.

## Verification
- **Never mask a command's exit code with `| tail`/`| head`** — the pipeline returns the pager's exit, hiding failures. Redirect to a log + `echo "EXIT=$?"`, or use `${PIPESTATUS[0]}`. (Bit us once: a failed build reported exit 0.)
- Hydration-safe "mounted" guard: use `useSyncExternalStore(noopSubscribe, () => true, () => false)`, NOT `useEffect(()=>setState(true))` — the latter trips `react-hooks/set-state-in-effect`.
